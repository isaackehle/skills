# Gap Answers

Scripted responses for common technical stack gaps. Lead with what you have, bridge to the gap — never apologize for it.

## Stack Gap Scripts

| Tool / Tech | Scripted Answer |
|-------------|----------------|
| NestJS | "I've built extensively in Node.js and Express — NestJS is the opinionated framework layer on top. I'd be productive quickly and have been evaluating it." |
| Redis / Kafka | "I've worked with event-driven architectures via SNS/SQS — I'd evaluate Redis or Kafka as the right tool where the use case warrants it." |
| Databricks / Fivetran / PostHog | "I've designed the data service layers these tools plug into. Familiar with the category, not yet hands-on." |
| LiteLLM / Amazon Bedrock | "I'm actively evaluating both for multi-provider LLM routing and enterprise-scale model access." |
| Go / Golang | "I haven't shipped in Go, but I have the systems background to ramp. The real question is how much of the day-to-day work is Go versus adjacent work." |
| Kubernetes (deep) | "I've worked with containerized deployments on ECS and have used Kubernetes at the surface level — orchestration concepts are solid, operator-level depth is the gap." |
| GraphQL (advanced) | "I've consumed GraphQL APIs and written basic schemas — deeper resolver and federation patterns would be the learning curve." |

## Usage Rules

- Lead with what you genuinely have — don't lead with the gap
- Frame gaps as "adjacent and evaluating" or "systems background transfers" — not as deficiencies
- If the gap is central to the role and not bridgeable, flag it honestly under Technical Fit in the scorecard
- Do not claim hands-on proficiency in tools you've only evaluated or read about
